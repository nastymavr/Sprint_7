# Тестирование API курьера и заказов

Этот проект представляет собой набор автоматизированных тестов для тестирования API сервиса курьеров и заказов. Тесты покрывают следующие аспекты:

- Создание курьера.
- Логин курьера.
- Создание заказа.
- Получение списка заказов.
- Генерация отчета Allure для отображения результатов тестов.

## Описание тестируемых ручек

### 1. **Создание курьера**
- **Курьер можно создать**: Проверка на создание курьера с уникальными данными.
- **Нельзя создать двух одинаковых курьеров**: Проверка, что при попытке создать курьера с уже существующим логином, API вернет ошибку.
- **Обязательные поля**: Тест на проверку обязательных полей при создании курьера (например, логин, пароль, имя).
- **Правильный код ответа**: Проверка, что успешный запрос возвращает статус код 201.
- **ok: true**: Проверка, что успешный запрос возвращает `ok: true`.
- **Ошибка при отсутствии поля**: Если одно из обязательных полей отсутствует, сервер возвращает ошибку.
- **Ошибка при существующем логине**: При попытке создать курьера с уже занятым логином возвращается ошибка.

### 2. **Логин курьера**
- **Авторизация курьера**: Проверка, что курьер может успешно авторизоваться с правильными данными.
- **Все обязательные поля для авторизации**: Тест на проверку, что для авторизации необходимо указать все обязательные поля (логин и пароль).
- **Ошибка при неверном логине или пароле**: Тест на проверку, что при ошибке в логине или пароле сервер возвращает ошибку.
- **Ошибка при отсутствии поля**: Тест на проверку, что если какое-либо обязательное поле отсутствует, то запрос вернет ошибку.
- **Авторизация несуществующего пользователя**: Тест на проверку, что если логин не существует, возвращается ошибка.
- **id в ответе**: Тест на проверку, что успешная авторизация возвращает `id` пользователя.

### 3. **Создание заказа**
- **Цвет заказа**: Проверка, что можно указать один из цветов (`BLACK` или `GREY`), оба цвета или не указывать цвет.
- **Тело ответа содержит track**: Проверка, что в теле ответа на создание заказа содержится `track`.
- **Параметризация**: Тесты создания заказа с разными вариантами цветов.

### 4. **Список заказов**
- **Получение списка заказов**: Проверка, что при запросе на получение списка заказов возвращается список заказов в теле ответа.

### 5. **Отчет Allure**
- Все тесты сопровождаются отчетом Allure, который генерируется после выполнения тестов для наглядного отображения результатов.

## Технологии

- **JUnit** - для написания тестов.
- **RestAssured** - для отправки HTTP-запросов.
- **Allure** - для генерации отчетов о тестах.
- **Java** - основной язык программирования.

## Установка и запуск

1. Клонируйте репозиторий:

    ```bash
    git clone <url-репозитория>
    cd <папка-проекта>
    ```

2. Убедитесь, что у вас установлены следующие зависимости:

    - JDK 11 и выше
    - Maven
    - IntelliJ IDEA (или любая другая IDE)

3. Установите все необходимые зависимости с помощью Maven:

    ```bash
    mvn clean install
    ```

4. Для запуска тестов используйте команду:

    ```bash
    mvn clean test
    ```

5. Для генерации отчета Allure выполните команду:

    ```bash
    mvn allure:serve
    ```

   После этого откроется отчет в браузере.

## Структура проекта

- `src/test/java` - директория с тестами.
- `src/main/java` - (если есть) основная логика проекта.
- `pom.xml` - файл конфигурации Maven для зависимостей и плагинов.

## Примечания

- Для запуска отчетов Allure необходимо установить плагин Allure в вашу систему.
- Все тесты написаны с использованием JUnit и RestAssured, для организации логики тестов используются аннотации JUnit и шаги Allure для более наглядного отображения.
